# 일반 시뮬레이션 시스템 (General Simulation System)

이 모듈은 수백, 수천, 수만 명의 투자자가 모였을 때의 상황을 시뮬레이션하는 시스템을 제공합니다. 각 투자자는 개별 회차(individual_round)를 가지며, 회사 회차(company_round)에 따라 움직이게 됩니다.

## 시스템 구조

일반 시뮬레이션 시스템은 다음과 같은 구조로 설계되었습니다:

```text
general/
  ├── models/
  │   ├── investor.py       - 투자자 모델 (GeneralInvestor)
  │   └── results.py        - 시뮬레이션 결과 모델 (GeneralSimulationResults, CompanyRoundResult)
  ├── services/
  │   └── general_simulator.py - 시뮬레이션 로직 (GeneralSimulationService)
  ├── utils/
  │   └── reporting.py      - 결과 보고 및 시각화 유틸리티
  └── main.py               - 메인 실행 스크립트
```

## 실행 방법

명령줄에서 다음과 같이 실행할 수 있습니다:

```bash
python -m src.python.general.main --plan A --investors 1000 --rounds 100 --plot --excel
```

### 매개변수

- `--plan`: 시뮬레이션할 투자 플랜 (기본값: "A")
- `--investors`: 시뮬레이션할 투자자 수 (100, 1000, 10000 중 선택, 기본값: 1000)
- `--rounds`: 시뮬레이션할 최대 회사 회차 수 (기본값: 100)
- `--output`: 결과물을 저장할 디렉토리 (기본값: "output")
- `--plot`: 결과를 시각화하여 그래프로 저장
- `--excel`: 결과를 Excel 파일로 저장

## 주요 기능

1. **다수의 투자자 시뮬레이션**: 수백, 수천, 수만 명의 투자자를 동시에 시뮬레이션
2. **납입금 흐름 분석**: 투자자가 납부한 금액이 다른 투자자의 수익으로 어떻게 흘러가는지 분석
3. **지속가능성 평가**: 시스템이 지속가능하기 위해 필요한 신규 가입자 수와 납입금 증가율 계산
4. **ROI 분포 분석**: 투자자별 ROI(투자 수익률) 분포 및 통계 제공
5. **종합적 결과 시각화**: 주요 지표를 그래프와 차트로 시각화하여 전체적인 시스템 건전성 평가

## 주요 질문에 대한 답변

일반 시뮬레이션은 다음 세 가지 주요 질문에 초점을 맞춰 설계되었습니다:

1. **납입금의 분배 과정**: 납입금이 어떻게 다른 투자자의 수익으로 분배되는지
2. **필요 납입금 증가율**: 모든 가입자가 안정적으로 수익을 가져가기 위해 필요한 회차당 납입금 증가율
3. **필요 신규 가입자 증가율**: 모든 가입자가 안정적으로 수익을 가져가기 위해 필요한 회차당 신규 가입자 증가율

## 결과 해석

시뮬레이션 결과에서 특히 다음 지표들에 주목하세요:

- **첫 흑자 전환 회차**: 시스템이 처음으로 누적 순익이 양수가 되는 회차
- **지속적인 흑자 유지 여부**: 흑자 전환 후 적자로 다시 돌아가지 않고 지속적으로 흑자를 유지하는지 여부
- **지속가능한 증가율**: 시스템이 지속가능하기 위해 필요한 신규 투자자 및 납입금 증가율
- **ROI 분포**: 투자자들의 수익률이 고르게 분포하는지, 일부에게 편중되는지 여부
- **수익/납입금 비율**: 회차별로 총 수익이 납입금을 얼마나 초과하는지를 보여주는 지표
