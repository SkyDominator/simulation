### 2. Research Result
**File:** `docs/research/PR-74/research-01.md`

**Key Findings:**
- Mobile viewports (Pixel 5: 393×851, iPhone 12: 390×844) are **portrait by default**
- `LandscapeEnforcer` correctly detects portrait orientation and shows blocking overlay
- `window.matchMedia("(orientation: portrait)").matches` returns `true` on portrait viewports
- Potential race condition: `initE2EMode()` sets flag via `page.addInitScript()`, but React component may initialize before flag is set
- Desktop tests unaffected (landscape viewports by default)

**Code Map:**
- Test infrastructure (initE2EMode, test helpers)
- LandscapeEnforcer component implementation
- E2E mode detection (isE2EMode)
- Playwright configuration (device presets)

**Recommended Solutions:**

2. Improve E2E mode detection reliability
