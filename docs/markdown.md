# GitHub Copilot Instructions for Editing Markdown Files

Write or edit markdown files according to the following process:

1. Write or edit the markdown file as per the requirements.
   * Read `Editing Checklist` below and ensure all items are checked.
   * If some items are not checked, fix them before proceeding.
2. After finishing the markdown file, read `Quality Assurance Checklist` below and ensure all items are checked.
    * If some items are not checked, fix them before proceeding.
3. Review the entire markdown file against `Editing Checklist` and `Quality Assurance Checklist` again to ensure all items are checked.
    * If some items are not checked, fix them before proceeding.
4. Repeat steps 1 to 3 until all items in all checklists are checked.
5. Submit the markdown file.

## Editing Checklist

### Direct Changes Only

If you are modifying the existing documents, do not use wordings that "showing there has been changes". For example, dont use wordings like these: "(Added part)" or "New sections". JUST CHANGE the document if there needs to be change.

### Consistent to Existing Style

If you are modifying the existing documents, fits the new or modified contents to the existing style, format, tone, and structure of the document, unless you are instructed otherwise.

## Quality Assurance Checklist

### Web Search for Verification

After finishing a markdown file, run web search to check all the commands, parameters, syntax, and all the methodologies in it are not deprecated as to the latest. If found, update any outdated commands or syntax based on your findings.

### Consistency Check

Ensure all contents are consistent with each other. They should not contradict each other or there should be no ambiguity.

### Redundancy Check

Ensure no redundancy. If there are redundant contents, consolidate them into one and remove the others.
