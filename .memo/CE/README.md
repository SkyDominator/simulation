# Spec driven development (SDD) for Brown field apps

## SDD: Basic flow

1. Write Spec doc
    1. Write the draft from codebase
    2. Review the draft with the latest best practices
    3. Review against the codebase
    4. Finalize the spec doc
2. Set up research questions
    1. Identify gaps between spec and codebase
    2. Prioritize the research questions
3. For each research question, write research doc
4. Run research and get results 
5. For each research result, set up implement plan doc
6. For each plan, implement the changes

## Write Spec doc

1. Generate spec from existing codebase
  * The spec must fits only the "existing requirements" (not future features, only for current development/production environment, only for the number of users we have now or planned in near future)

2. Review the spec with the latest best practices
  * Compare with the best practices and identify gaps

3. Review the spec against the codebase
